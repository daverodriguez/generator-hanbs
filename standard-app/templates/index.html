How to use HanBootStrap (HanBS)

Use the main.js file that's loaded along with this page as the starting point of your application. Main.js has one dependency, HanBootStrap (<%= jsPath %>/lib/hbs.js). HanBS includes some object-creation and extension functions to make object-oriented Javascript easier. It also contains a function that runs when the page is loaded and looks for two data variables on the body element (data-section and data-page).

<body data-section="<%= appNS %>.sections.<%= controllerName %>" data-page="examplePage">

If you want to auto-load and init your Javascript, do this:

<body data-section="<%= appNS %>.sections.<%= controllerName %>" data-page="examplePage"
						data-autoload data-script-async="<%= jsPath %>/<%= appNS %>/controllers/<%= controllerName %>.js">

Instead of maintaining separate include files for each section, we use these two variables to load a section-specific Javascript file and run a specific function to initialize the current page.

The bootstrapper first attempts to load the Javascript class referenced by data-section. That class must be loaded separately in a script tag or by some other method (or turn on autoload?).

This file is called a section controller. It contains an init() function that holds code that needs to run on every page in a section, and can optionally contain a specific function for every page. The init() function is automatically run if it exists.

After the section controller loads and init() is run, the bootstrapper looks for a function in the section controller matching the data-page attribute, in this case myPage(). If that function exists, it is run.

A section controller or might load many other Javascript classes and/or third party libraries.
